getStatusUserProduct()

InvestmentProductController.php : 146

Retrieve the status of a user's investment product.

public getStatusUserProduct(int $productId) : JsonResponse
Parameters
$productId : int

The ID of the investment product.

Return values
JsonResponse —

The JSON response containing the status of the user's investment product.

This method performs the following steps:

  1. Retrieves the authenticated user's ID.
  2. Queries the 'investment_product' table to find the latest status of the user's investment product.
  3. If the user's product is not found, returns a JSON response with status 'none'.
  4. If the user's product is found: a. Retrieves all tokens for the user's investment product using InvestmentsHelper. b. Calculates the sum of payments with statuses 'PENDING', 'REQUESTED', or 'COMPLETED' for the user's investment product. c. Adds the completed tokens, pending amount, and payments to the user product object. d. Returns the user product as a JSON response.

public savesingle(Request $request) : RedirectResponse|Response

Validates the request data to ensure the amount is numeric and greater than zero, and that the product_id exists in the products table. If validation fails, it redirects back with errors.

Retrieves the authenticated user's ID and the default coin setting. Fetches the product with its associated pool based on the provided product_id. If the product is not enabled for new investments, it redirects back with an error.

Processes an investment for a user.

protected processInvestment(int $userId, object $product, float $requestAmount, object $defaultCoin) : array<string|int, mixed>

This method performs several checks and operations to process an investment:

  • Validates the investment amount against the product's minimum and maximum limits.
  • Verifies the user's available balance.
  • Initiates a database transaction to create the investment product.
  • Handles pending payments if the user's available balance is insufficient.
  • Dispatches jobs to verify payments and process withdrawals.
  • Sends an email notification to the user.
