accountVerificationPersonDocumentation()

AccountVerificationControllerv2.php : 1015

Handles the account verification process for a person.

public accountVerificationPersonDocumentation() : RedirectResponse|Response

This method performs the following steps:

  1. Retrieves the authenticated user's ID.
  2. Logs the initiation of the account verification process.
  3. Checks if the account verification process is already completed and redirects if so.
  4. Attempts to retrieve an incomplete account verification record for the user.
  5. If an account verification record is found:
    • Logs the retrieval of the account verification record.
    • Initializes the AliceHelper and retrieves the application environment.
    • Performs flow validation and risk verification using AliceHelper.
    • Logs the results of the validations.
    • Saves the validation results in the account verification record.
    • If flow validation is successful:
      • Performs AML verification if the environment is 'Alldefi'.
      • If AML verification is successful:
        • If risk verification is successful:
          • Updates the account verification status to 'VERIFIED'.
          • Updates the user's KYC status to 'VERIFIED' and risk level to 'MEDIO'.
          • Adds the user to the monitoring list.
          • Dispatches a process to handle the completion of the account validation.
          • Sends a notification to Discord and logs the completion.
        • If risk verification fails:
          • Updates the account verification status to 'COMPLETED'.
          • Updates the user's KYC status to 'COMPLETED' and risk level to 'ALTO'.
          • Dispatches a process to handle the account validation.
          • Sends a notification to Discord and logs the high risk incident.
          • Saves the incident in the account verification record.
      • If AML verification fails:
        • Updates the account verification status to 'COMPLETED'.
        • Updates the user's KYC status to 'COMPLETED' and risk level to 'ALTO'.
        • Dispatches a process to handle the account validation.
        • Sends a notification to Discord and logs the AML risk incident.
        • Saves the incident in the account verification record.
    • If flow validation fails:
      • Updates the user's KYC status to 'COMPLETED' and risk level to 'ALTO'.
      • Sends a notification to Discord and logs the unsuccessful response report.
      • Saves the incident in the account verification record.
    • Renders the account verification success page with the success status.
  6. If no account verification record is found, redirects to the account verification route.
  7. Catches and logs any exceptions that occur during the process.
  8. Redirects to the account verification route in case of an error.
Return values
RedirectResponse|Response

getZipcode()

Retrieve the zipcode from a given address using Google Maps Geocoding API.

public getZipcode(string $address) : string|false
Parameters
$address : string

The address to retrieve the zipcode for.

Return values
string|false —

The zipcode if found, or false if not found or an error occurs.

This function performs the following steps:

  1. Formats the address by replacing spaces with '+'.
  2. Sends a request to the Google Maps Geocoding API to get the latitude and longitude of the address.
  3. Sends another request to the Google Maps Geocoding API to get the address components using the latitude and longitude.
  4. Extracts and returns the zipcode from the address components if available.
  5. Logs various information and errors for debugging purposes.

Note: Ensure that you replace the API key in the URL with a valid Google Maps API key.
